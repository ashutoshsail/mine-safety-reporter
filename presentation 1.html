Mine Safety Reporter

A Proposal for a New PWA

Seeking Partnership for Deployment

The Problem: A Fragmented System

▶️ Reporting is a manual, decentralized process.

  • Lack of Standardization: Formats vary widely (PDF, Excel, handwritten).
  • Data Loss Risk: Records on local PCs are vulnerable to crashes.
  • Ineffective Follow-up: Central safety lacks visibility on incident status.
  • Poor Reporting Culture: On-site infrastructure discourages immediate reporting.

The Solution: A Modern, Secure PWA

▶️ The application provides a centralized, digital platform.

  • Standardized Data: A single format for all reports on a secure server.
  • Real-time Visibility: Live dashboards provide instant updates across all mines.
  • Mobile-First Reporting: Enables on-site personnel to report near-misses instantly.
  • Proactive Insights: Structured data allows for trend analysis.

Core Functionality Overview

▶️ Key features include:

  • Incident Reporting: A multi-step form to capture all required details.
  • Incident Log: A filterable log of reports with detailed views and PDF export.
  • Analytics Dashboards: Visual tools for management to analyze performance and trends.
  • Admin Panel: An interface to manage key application data (mines, sections, users).

Technical Architecture

▶️ The app is built on a modern, robust stack.

  • Frontend: React PWA with Vite.
  • Styling: Tailwind CSS for a professional UI.
  • Backend: Firebase for secure and scalable cloud services.
  • Data Layer: React Context API for global state management.

Security & Data Integrity

▶️ The app ensures security and accountability through:

  • Custom Login: Uses User IDs with Firebase Auth.
  • Immutable Audit Log: Every change is logged with a timestamp.
  • Role-Based Access: The Admin Panel is restricted to authorized users.
  • Centralized Storage: Protects against local data loss.
